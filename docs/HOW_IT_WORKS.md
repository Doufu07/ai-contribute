# AI 贡献分析

`ai-contribute` 是一个用于分析统计本地项目 AI 贡献情况的工具。通过扫描各 AI 助手的本地会话日志，结合代码库现状，精确计算 AI 实际贡献的代码行数。

## 快速上手

### 1. 安装和运行

**方式一：直接使用 npx（推荐，无需安装）**
```bash
npx ai-contribute /path/to/your/project
```

**方式二：全局安装**
```bash
npm install -g ai-contribute
ai-contribute /path/to/your/project
```

### 2. 检查可用工具

运行以下命令查看当前环境检测到了哪些 AI 工具的数据：
```bash
npx ai-contribute list
```

## 常用命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `scan` | 扫描并生成统计报告 (默认) | `npx ai-contribute . -d src -v` |
| `list` | 列出检测到的 AI 工具 | `npx ai-contribute list` |
| `files` | 查看各文件 AI 贡献详情 | `npx ai-contribute files -n 20` |
| `history` | 查看 AI 贡献时间线 | `npx ai-contribute history` |
| `sessions` | 列出所有 AI 会话记录 | `npx ai-contribute sessions` |

### 实际示例

**基础分析**
```bash
npx ai-contribute .                    # 分析当前目录
npx ai-contribute /path/to/project     # 分析指定项目
```

**详细输出（包含目录、文件、时间线）**
```bash
npx ai-contribute . -v
```

**只分析 src 目录（适用于 Vue/React 项目）**
```bash
npx ai-contribute . -d src -v
```

**只分析特定 AI 工具**
```bash
npx ai-contribute . -t claude,codex
```

**导出报告**
```bash
npx ai-contribute . -f json -o report.json
npx ai-contribute . -f markdown -o report.md
```

**切换验证模式**
```bash
npx ai-contribute . --verification strict    # 严格匹配
npx ai-contribute . --verification historical # 信任原始数据
```

## 支持的 AI 工具

目前支持自动检测以下 AI 编码工具：

| 工具 | 默认数据位置 (macOS) | 自动检测 |
|------|----------|----------|
| **Claude Code** | `~/.claude/projects/` | ✓ |
| **Codex CLI** | `~/.codex/sessions/` | ✓ |
| **Cursor** | `~/Library/Application Support/Cursor/` | ✓ |
| **Gemini CLI** | `~/.gemini/tmp/` | ✓ |
| **Opencode** | `~/.local/share/opencode/` | ✓ |
| **Trae** | `~/Library/Application Support/Trae/` | ✓ |

**关键指标说明：**
- **原始数据 (Raw Data)**：项目当前的文件数和总行数（全量扫描时）。
- **新增代码 (Added Code)**：项目中的实际新增行数。全量扫描时等于总行数；指定时间范围（`--since`）时为该时间段内的新增行数。

- **AI贡献 (AI Contribution)**：经过验证后，确认由 AI 生成且保留在当前项目中的代码量（实际用了多少）。
- **AI生成占比 (AI Generation Share)**：`AI贡献 / 原始数据`（或新增代码），反映 AI 代码在项目中的渗透率。
- **采纳率 (Adoption Rate)**：`AI贡献 / AI生成`，反映 AI 生成代码的有效利用率。

## 工作原理

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  AI Session     │     │  Verify Lines   │     │  Generate       │
│  Logs           │ ──▶ │  Against Repo   │ ──▶ │  Statistics     │
│  (JSONL/JSON)   │     │  (Deduplicate)  │     │  Report         │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

1. **扫描 (Scan)**: 读取各 AI 工具的本地会话日志。
2. **解析 (Parse)**: 提取文件变更操作（新增行、删除行）。
3. **验证 (Verify)**: 检查提取的行是否仍存在于当前代码库中。
4. **去重 (Deduplicate)**: 确保同一行代码只被计数一次（即使在多个会话中出现）。
5. **统计 (Stats)**: 生成最终的贡献分析报告。

## 核心技术逻辑

`ai-contribute` 通过以下核心机制确保统计的准确性和公正性：

### 1. 整体架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           ai-contribute                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐          │
│  │ Scanners │ →  │ Analyzer │ →  │ Verifier │ →  │ Reporter │          │
│  └──────────┘    └──────────┘    └──────────┘    └──────────┘          │
│  (多工具适配)      (变更提取)       (存活验证)       (数据可视化)          │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2. 多源异构数据解析 (Scanners)

不同 AI 工具存储会话的方式各异，我们实现了专门的适配层：

- **Claude Code / Codex**: 解析 JSONL 格式的流式日志。
- **Cursor**: 解码 SQLite 数据库中的操作记录。
- **Trae**: 结合 SQLite (会话元数据) 和 Git Snapshots (代码变更快照) 进行混合分析。

### 3. 智能代码指纹与验证 (Verification)

仅仅统计 AI 生成的代码是不够的，我们必须确认这些代码**是否被最终采纳**。

1. **变更提取**：使用 LCS (最长公共子序列) 算法，从 AI 会话中精确提取出新增的代码行。
2. **存活验证**：
    - **严格模式**：要求代码行在当前文件中完全匹配。
    - **宽松模式 (默认)**：忽略空格和缩进差异，只要逻辑代码一致即视为匹配。
3. **全局去重**：如果多个 AI 会话（或多次对话）生成了相同的代码，系统会自动去重，避免重复计数。

### 4. 性能与安全

- **流式处理**：支持 GB 级别的超大会话日志文件，占用极低内存。
- **并行计算**：利用 Worker Threads 在后台线程进行繁重的 Diff 计算，保持界面流畅。
- **本地运行**：所有分析全在本地完成，**不会上传任何代码**到云端，确保数据安全。
